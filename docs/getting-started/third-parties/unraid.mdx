## Template installation

Several templates from Unraid Community are available right now
- [Unraid Community Apps](https://unraid.net/community/apps#community-apps-iframe)

In case you still want to do it manually, please follow the instructions below.

## Manual Installation

Before proceeding, choose which installation method best suits your server:

**Seerr Default** — Runs the container with its native `node` user (UID 1000). This method aligns with how the Seerr image is officially packaged and is suitable for most standard setups.

:::caution
The Seerr Default method sets appdata ownership to UID 1000:1000, which conflicts with Unraid's expectation that appdata is owned by `nobody:users` (UID 99, GID 100). This can break SMB share access (users won't be able to edit config files over the network), trigger warnings in the "Fix Common Problems" plugin, and cause issues with backup plugins (such as CA Appdata Backup) and network integrations (such as Tailscale).
:::

**Unraid Default** — Runs the container with Unraid's native `nobody:users` ownership (UID 99, GID 100). This method follows Unraid best practices and maintains native file ownership. Use this method if you need to retain standard SMB share access without permission errors, or if you are running specific Unraid network integrations (like Tailscale) that require specific routing.

:::caution
The Unraid Default method overrides the container's built-in user with `--user 99:100`. Because Seerr is designed to run as a rootless container under its native `node` user (UID 1000), overriding this may lead to unexpected permission issues or compatibility problems with future Seerr updates.
:::

<Tabs groupId="unraid-permissions" queryString>
  <TabItem value="seerr-default" label="Seerr Default">

  ### 1. Create the config directory

  :::note
  Seerr is now rootless. Unraid typically runs containers as `nobody:users` (UID 99, GID 100) while Seerr runs as UID 1000, GID 1000. This method changes the host folder ownership so Seerr runs with its native UID/GID.
  :::

  Open the Unraid terminal and run:

  ```bash
  mkdir -p /mnt/user/appdata/seerr
  ```

  Change ownership of the config folder on the host so Seerr runs with its native UID/GID:

  ```bash
  chown -R 1000:1000 /mnt/user/appdata/seerr
  ```

  ### 2. Add the Docker container

  :::warning
  The **Extra Parameters** field is critical. You **must** add `--init --restart=unless-stopped` (see table below). Without `--init`, the container will not handle signals properly or shut down cleanly. Without `--restart=unless-stopped`, the container will not automatically restart after a reboot.

  To see the **Extra Parameters** field, click **Basic View** in the top-right corner of the template page to switch to the advanced editor.
  :::

  Navigate to the **Docker** tab in Unraid and click **Add Container**. Fill in the following:

  | Field | Value |
  |---|---|
  | **Name** | `seerr` |
  | **Repository** | `ghcr.io/seerr-team/seerr:latest` |
  | **Icon URL** | `https://raw.githubusercontent.com/seerr-team/seerr/develop/public/android-chrome-512x512.png` |
  | **WebUI** | `http://[IP]:[PORT:5055]` |
  | **Extra Parameters** | `--init --restart=unless-stopped` |
  | **Network Type** | `bridge` |
  | **Privileged** | `Off` |

  Then click **Add another Path, Port, Variable** to add:

  **Port:**
  | Field | Value |
  |---|---|
  | Container Port | `5055` |
  | Host Port | `5055` |
  | Connection Type | `TCP` |

  **Path:**
  | Field | Value |
  |---|---|
  | Container Path | `/app/config` |
  | Host Path | `/mnt/user/appdata/seerr` |

  **Variable:**
  | Field | Value |
  |---|---|
  | Key | `TZ` |
  | Value | Your [TZ database name](https://en.wikipedia.org/wiki/List_of_tz_database_time_zones) (e.g., `America/New_York`) |

  **Variable (optional):**
  | Field | Value |
  |---|---|
  | Key | `LOG_LEVEL` |
  | Value | `info` |

  Click **Apply** to create and start the container.

  ### 3. Access Seerr

  Open the WebUI at `http://<your-unraid-ip>:5055` and follow the setup wizard.

  </TabItem>
  <TabItem value="unraid-default" label="Unraid Default">

  ### 1. Create the config directory

  :::note
  Keep current host ownership and run the container as Unraid's `nobody:users` (UID 99, GID 100). This avoids changing host permissions while allowing the container to access your shares.
  :::

  Open the Unraid terminal and run:

  ```bash
  mkdir -p /mnt/user/appdata/seerr
  ```

  Change ownership of the config folder on the host so Seerr runs with Unraid native UID/GID:

  ```bash
  chown -R 99:100 /mnt/user/appdata/seerr
  ```

  ### 2. Add the Docker container

  :::warning
  The **Extra Parameters** field is critical. You **must** add `--init --restart=unless-stopped --user 99:100` (see table below). Without `--init`, the container will not handle signals properly or shut down cleanly. Without `--restart=unless-stopped`, the container will not automatically restart after a reboot. The `--user 99:100` parameter runs the container process with Unraid's default UID/GID and avoids permission errors accessing your shares without changing host folder ownership.

  To see the **Extra Parameters** field, click **Basic View** in the top-right corner of the template page to switch to the advanced editor.
  :::

  Navigate to the **Docker** tab in Unraid and click **Add Container**. Fill in the following:

  | **WebUI** | `http://[IP]:[PORT:5055]` |
  | **Extra Parameters** | `--init --restart=unless-stopped --user 99:100` |
